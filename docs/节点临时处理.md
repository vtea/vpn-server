# 节点临时处理（WG + 节点连通）

## 1) 一键止血：修复同机 WG 端口冲突

控制面仓库提供脚本：`vpn-api/scripts/wg-owner-heal.sh`

在每台故障节点执行（会自动扫描本机 `wg-node-*`）：

```bash
sudo bash -c '
set -euo pipefail
curl -fsSL http://127.0.0.1:56700/api/node-setup.sh >/dev/null 2>&1 || true
bash /opt/vpn-api/vpn-api/scripts/wg-owner-heal.sh
'
```

默认策略：
- 仅一个 owner 接口固定 `ListenPort=51820`
- 其余 `wg-node-*` 删除 `ListenPort`（自动端口）

可手动指定 owner：

```bash
sudo bash /opt/vpn-api/vpn-api/scripts/wg-owner-heal.sh --owner wg-node-20
```

## 2) 一键排障：定位 sent>0 / received=0

脚本：`vpn-api/scripts/wg-link-triage.sh`

```bash
sudo bash /opt/vpn-api/vpn-api/scripts/wg-link-triage.sh
```

仅排查某个 peer：

```bash
sudo bash /opt/vpn-api/vpn-api/scripts/wg-link-triage.sh --peer node-20
```

## 3) 结果判读

- 若输出 `[ALERT] ... status=sent_only`：
  - 对端 `wg-quick@wg-node-<本节点ID>` 未启动或配置不一致；
  - 双向安全组/防火墙未放行 UDP 51820；
  - `peer_endpoint / peer_pubkey / allowed_ips` 仍不一致。

- 若 `latest handshake` 持续刷新且 `rx/tx` 双向增长：
  - 链路恢复。

## 4) 10 分钟稳定性验收

脚本：`vpn-api/scripts/wg-e2e-verify.sh`

```bash
sudo bash /opt/vpn-api/vpn-api/scripts/wg-e2e-verify.sh --minutes 10
```

仅验某个 peer：

```bash
sudo bash /opt/vpn-api/vpn-api/scripts/wg-e2e-verify.sh --minutes 10 --peer node-20
```

## 5) 说明

- “同网段不同机器都使用 51820”不是问题；
- 真正冲突是“同一台机器多个 WG 接口同时绑定同一个本地端口”。
