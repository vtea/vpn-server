# 用户使用指南

## 获取配置文件

管理员会为你生成一个 `.ovpn` 配置文件并发送给你。这个文件包含了连接 VPN 所需的所有信息。

### TCP 与 UDP 两份配置

控制面会为同一用户证书同时提供 **TCP** 与 **UDP** 两种客户端配置（仅首部 `proto` 不同，证书相同）。请按 VPN **节点实际监听的协议**选用：若服务端只开了 TCP，则只有 TCP 版能连通；只开了 UDP 则只有 UDP 版能连通。不确定时可向管理员确认，或先试与管理员告知的协议一致的那份。

「默认」下载一般对应控制面为该实例登记的协议，与节点现场一致时最省事。

## 安装 OpenVPN 客户端

| 平台 | 推荐客户端 | 下载地址 |
|------|-----------|---------|
| Windows | OpenVPN Connect | https://openvpn.net/client/ |
| macOS | Tunnelblick | https://tunnelblick.net/ |
| iOS | OpenVPN Connect | App Store 搜索 "OpenVPN Connect" |
| Android | OpenVPN Connect | Google Play 搜索 "OpenVPN Connect" |
| Linux | 命令行 openvpn | `sudo apt install openvpn` |

## 导入配置文件

### Windows / macOS / 手机
1. 打开 OpenVPN Connect
2. 点击 "Import" 或 "+" 按钮
3. 选择管理员发给你的 `.ovpn` 文件
4. 点击 "Connect"

### Linux
```bash
sudo openvpn --config your-config.ovpn
```

## 流量模式说明

管理员可能为你分配了不同的配置文件，对应不同的流量模式（同一节点可能属于多个**组网网段**，每套模式有独立子网与端口）：

| 配置文件名中的关键词 | 含义 | 适用场景 |
|---|---|---|
| `local-only` | 本节点出口（可选对端出口） | 默认：**公网流量经你连接的这台 VPN 节点**出网（像传统 VPN）。管理员也可为该模式指定**出口节点**，则流量经隧道到另一台机器再出网 |
| `hk-smart-split` | 香港智能分流 | 日常办公：中国网站直连，海外网站走香港 |
| `hk-global` | 香港全局代理 | 所有流量走香港出口（或管理员指定的出口节点） |
| `us-global` | 美国全局代理 | 所有流量走美国出口（或管理员指定的出口节点） |

**`local-only` 与「上网」（当前版本）**：服务端一般会推送 **默认路由走 VPN**，公网流量经 **VPN 服务器一侧的出口**出去。若你仍使用**旧版节点脚本**生成的配置，可能仍是「仅 VPN 子网进隧道」的旧行为，表现为连上后公网仍走本机 Wi‑Fi/4G；此时可请管理员升级节点并**重跑安装脚本**。若你开了 **「仅允许 VPN / Kill Switch」** 却仍上不了网，检查是否拿到带默认路由的配置，或暂时关闭该选项排查。

**智能分流模式**对用户完全透明——你不需要做任何设置，系统会自动判断目标网站是国内还是海外，选择最优路径。

## 常见问题

### Q: 连接后无法上网？
A: 先确认分配的是哪种模式。当前 **`local-only` 预期为经 VPN 节点出公网**；若管理员尚未升级节点脚本，可能仍是旧行为（公网不走隧道），需升级并重装节点侧配置。`hk-global` / `us-global` / `hk-smart-split` 用于香港/美国分流或指定出口。若已确认配置较新仍异常，检查客户端 **Kill Switch / 仅 VPN**、DNS 与防火墙。

### Q: 连接速度很慢？
A: 如果使用全局代理模式，所有流量都经过海外节点，访问国内网站会变慢。建议使用 `smart-split`（智能分流）模式。

### Q: 提示证书错误？
A: 配置文件可能已被管理员吊销。请联系管理员重新生成。

### Q: 能同时连接多个配置吗？
A: 不能。同一时间只能使用一个 VPN 配置。如需切换，请先断开当前连接。

### Q: 手机上如何使用？
A: 管理员发给你的 `.ovpn` 文件可以通过微信/邮件发送到手机，然后用 OpenVPN Connect App 打开导入。

### Q: 连接经常断开？
A: 检查网络环境。如果在不稳定的 WiFi 下，建议切换到移动数据。OpenVPN 客户端通常会自动重连。
