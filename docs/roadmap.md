# 实施路线图 — 按优先级分阶段执行

> 总计约 10 周，每阶段有明确交付物和验收标准。
> 详细技术方案见 [architecture.md](./architecture.md)

---

## P0 — 基础规划（第 1 周）

**目标**：定稿地址规划，准备测试环境，确认技术选型无阻塞项。

| # | 任务 | 产出 | 验收标准 |
|---|------|------|----------|
| 0.1 | 确认所有节点的公网 IP 和地域 | 节点清单表 | 6 个节点信息齐全 |
| 0.2 | 定稿 OpenVPN 子网编码（10.X.Y.0/24） | 地址规划表 | 无重叠，预留充足 |
| 0.3 | 定稿 WireGuard 骨干隧道地址（172.16.0.0/24） | 隧道分配表 | 所有节点对已分配 |
| 0.4 | 准备 2 台测试服务器（模拟上海+香港） | 可 SSH 登录 | OS 已安装，公网可达 |
| 0.5 | 确认控制面部署位置（云主机 or 节点上） | 部署方案 | 有明确的机器和域名 |
| 0.6 | 注册域名或配置内网 DNS（如 vpn-api.company.com） | DNS 记录 | 可解析 |

**依赖**：无，可立即开始。

## P2 — 多实例 + 分流 + 部署脚本（第 4-5 周）

**目标**：试点节点跑通三种流量模式，一键脚本可部署新节点。

### P2.1 多 OpenVPN 实例（第 4 周）

| # | 任务 | 产出 | 验收标准 |
|---|------|------|----------|
| 2.1 | 在上海测试机部署 node-direct（节点直连）实例 | server.conf + systemd + NAT | 客户端连接后可上网，IP 为上海公网 |
| 2.2 | 部署 global（全局）实例 | server.conf + 路由表 102 | 客户端连接后所有流量走指定出口 |
| 2.3 | 部署 cn-split（国内分流）实例 | server.conf + ipset + 路由表 101 | 访问百度走本地，访问 Google 走出口 |
| 2.4 | 在香港测试机配置出境 MASQUERADE | iptables 规则 | 从上海转发来的流量能正确 NAT 出去 |
| 2.5 | 验证证书隔离（不同实例的证书不能互连） | 测试报告 | cn-split 的证书连不上 node-direct |

### P2.2 Node Agent 第一版（第 4 周）

| # | 任务 | 产出 | 验收标准 |
|---|------|------|----------|

| 2.7 | Agent 证书操作（签发 + 吊销 + 生成 .ovpn） | agent/ovpn_manager.go | API 发指令后 Agent 返回 .ovpn 文件 |

### P2.3 一键部署脚本（第 5 周）

| # | 任务 | 产出 | 验收标准 |

### P2.4 授权流程打通（第 5 周）

| # | 任务 | 产出 | 验收标准 |

## 优先级排序总结

```
紧急且重要（先做）          重要但不紧急（后做）
┌─────────────────────┐   ┌─────────────────────┐
│ P0 地址规划          │   │ P3 Web 管理端        │
│ P1 API + WireGuard   │   │ P3 IP 库自动更新     │
│ P2 多实例 + 分流验证  │   │ P4 全面上线          │
│ P2 部署脚本          │   │ P4 运维文档          │
│ P2 授权流程          │   │                     │
└─────────────────────┘   └─────────────────────┘

不紧急不重要（可选）
┌─────────────────────┐
│ 隧道自动故障切换     │
│ 出境节点负载均衡     │
│ 员工自助门户         │
│ 短期证书             │
│ Prometheus/Grafana   │
└─────────────────────┘
```
