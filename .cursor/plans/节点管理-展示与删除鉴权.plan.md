# 节点管理：仅展示已启用接入、操作列编辑/删除、删除需密码

## 需求（本次讨论补充）

1. **未启用的接入（实例）不在列表中展示**  
   - **节点列表页**「接入/网段」列：当前遍历 `row.instances` 展示全部模式；应**仅展示 `enabled === true`** 的实例标签。  
   - **节点详情页**「组网接入」表：当前展示全部实例并带开关；应**默认仅展示 `enabled === true` 的行**（与截图中希望隐藏未启用行一致）。

2. **操作列：编辑 + 删除**  
   - **节点列表**「操作」列：在现有「删除」基础上增加 **「编辑」**（[`Nodes.vue`](openvpn-install-master/vpn-admin-web/src/views/Nodes.vue)）。  
   - **编辑**行为：与名称链路等价，进入 **`/nodes/:id` 详情页**即可（无需新页面时）；若后续支持改名称/地域/公网 IP，再增加 `PATCH /api/nodes/:id`（**当前 API 无更新节点**，见下）。

3. **删除节点需验证管理员密码**  
   - 防止误触：删除前弹出输入框，提交**当前登录管理员密码**，校验通过后再调用删除接口。  
   - **后端**：现有 [`DELETE /api/nodes/:id`](openvpn-install-master/vpn-api/cmd/api/main.go) **无密码校验**；需扩展为例如：  
     - **推荐**：`POST /api/nodes/:id/delete`（或 `DELETE` 携带 JSON body，部分客户端对 DELETE body 支持差）Body：`{"password":"..."}`，从 JWT 取 `sub`，`bcrypt.CompareHashAndPassword` 与库中 `Admin.PasswordHash` 比对，失败返回 `401/403`。  
   - **前端**：删除改为自定义 `el-dialog` + 密码输入，不再仅用 `el-popconfirm`。

---

## 仅展示启用行时的交互缺口（需定稿）

若详情页主表**完全隐藏** `enabled === false` 的实例，用户将无法在页面上**重新启用**某模式。请在实现时二选一或组合：

- **A**：主表仅 `enabled`；下方增加折叠区 **「已禁用的接入 (N)」**，内嵌简表 + 开关，与现逻辑一致。  
- **B**：主表仍显示全部，但**未启用行用折叠/次要样式**（与「完全不显示」冲突，若坚持不显示则用 A）。

本计划默认采用 **A**：主表仅启用；折叠区管理已禁用实例。

## 执行清单（实现阶段）

1. **vpn-admin-web `Nodes.vue`**：`v-for` 改为仅 `row.instances.filter(i => i.enabled)`；空时展示「暂无」或「无已启用接入」；操作列增加「编辑」跳转 `/nodes/:id`；删除改为密码对话框，请求新接口或带密码的删除接口。  
2. **vpn-admin-web `NodeDetail.vue`**：主表 `instances` 过滤为 `enabled`；增加「已禁用的接入」折叠区列出 `!enabled` 并保留开关。  
3. **vpn-api**：实现带密码校验的删除（新路由 + 从 JWT 取管理员并验密码）；必要时废弃裸 `DELETE` 或内部复用同一校验函数。  
4. **文档**：[`vpn-api/README.md`](openvpn-install-master/vpn-api/README.md) 或操作说明中增加删除接口说明。
