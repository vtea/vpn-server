## 一、问题与结论（按主题）

### 1. 浏览器控制台 `content_script.js`

- **性质**：浏览器扩展注入脚本，**与项目代码无关**。
- **处理**：无需改代码；排查接口问题时以 Network / 后端日志为准。

### 2. 接口 500、`AxiosError`

- **含义**：请求已到达 `vpn-api`，但处理失败（常见：SQLite、迁移、业务错误）。
- **已做（若已合并）**：响应体 `error` 优先展示；开发环境 `console.error('[vpn-api]', …)`；`ListExceptions` 与列表类接口一致返回 DB 错误；SQLite 连接串增加 `busy_timeout`；部分页面 `mounted` 请求加 `catch` 避免未处理 Promise。
- **待完善**：根因需结合**当时**响应 JSON 与**服务端终端日志**个案分析；若仍有空 `error`，可再加强 Gin 侧日志。

### 3. 登录转圈 / Vite `EADDRINUSE`、`ENOBUFS`、代理连错

- **根因**：`9901` 被占用时，旧版 Vite 会**自动改用 9902**，与 **vpn-api 默认 9902** 冲突，`/api` 代理指向了 Vite 自身。
- **已做（若已合并）**：`vite.config.js` 中 `strictPort: true`；`package.json` 增加 `dev:5173`；`docs/build-and-run.md` 说明。
- **操作要点**：先起 **vpn-api:9902**，再起 **Vite:9901**；占用 9901 时结束旧 `node` 进程或使用 `npm run dev:5173`。

### 4. 首次登录后菜单不全、角色「未知」

- **根因**：JWT 第二段为 **base64url**，用 `atob` 直接解**会失败**，`role`/`perms` 读不到，`hasPerm` 全 false。
- **已做（若已合并）**：`src/utils/jwt.js` 中 `parseJwtPayload`；`App.vue`、`Admins.vue` 改用该函数。

### 5. 节点 ID 与同名冲突

- **根因**：`BuildNodeID(name)` 仅从名称 slug，同名会得到相同 `id`。
- **拟定方案**：`NodeIDFromNumber(num)`（如 `node-{node_number}`），在 `CreateNode` 中在取得 `num` 后赋值；**删除** `BuildNodeID(name)`。
- **状态**：见下方「待代码执行」。

### 6. 添加节点时无「内网 IP」表单项

- **结论**：**刻意设计**。`10.x` VPN 子网由 **`BuildInstancesForMembership`** 按 `node_number`、网段 `SecondOctet`、`slot`、模式序号**自动计算**；**公网 IP** 仅表示站点可达性。节点通过 Agent **拉取实例与隧道配置**获知 `subnet`/端口/WG 地址。
- **待完善（体验）**：可在「添加节点」弹窗增加**一句说明文案**（可选，见 [节点id与地址唯一性.plan.md](./节点id与地址唯一性.plan.md)）。

### 7. 部署后地址唯一、多节点组网可靠

- **结论**：default 下依赖 **`node_number` 唯一**；非 default 依赖**网段内槽位递增**；WG `/30` 依赖**隧道顺序分配**。详见 [节点id与地址唯一性.plan.md](./节点id与地址唯一性.plan.md)。
- **待完善（高并发/Postgres）**：`NextNodeNumber` 在事务外、`tunnels.subnet` 无唯一索引等——文档中已列**可选加固**，按环境择一实施。
